# 보안 정책

[English](../SECURITY.md) | **한국어**

## 지원 버전

| 버전 | 지원 여부          |
|------|--------------------|
| 최신 | :white_check_mark: |

rdprrap은 활발히 개발 중이며, 보안 업데이트는 `main` 브랜치의 최신 버전에 적용됩니다.

## 취약점 제보 방법

**공개 GitHub 이슈를 통해 보안 취약점을 제보하지 마세요.**

대신 [GitHub Security Advisories](https://github.com/kernalix7/rdprrap/security/advisories/new)를 통해 제보해 주세요.

### 제보 시 포함할 내용

1. **설명** — 취약점에 대한 명확한 설명
2. **재현 방법** — 이슈를 재현하는 상세한 단계
3. **영향** — 취약점의 잠재적 영향
4. **영향 받는 컴포넌트** — rdprrap의 어떤 크레이트/DLL이 영향을 받는지
5. **환경** — Windows 버전, 아키텍처 (x64/x86), termsrv.dll 버전

### 대응 일정

- **수신 확인** — 제보 후 48시간 이내
- **초기 평가** — 7일 이내
- **수정 및 공개** — 제보자와 협의; 중요 이슈는 통상 30일 이내

### 범위 (In Scope)

- `unsafe` 블록의 메모리 안전성 위반
- DLL 하이재킹 / 로딩 경로 취약점
- PE 파싱 또는 디스어셈블리의 버퍼 오버플로우
- 스레드 일시정지/재개의 경쟁 조건
- WriteProcessMemory의 잘못된 주소 타겟팅
- 래퍼 DLL을 통한 권한 상승

### 범위 외 (Out of Scope)

- 사용자 머신에 물리적 접근이 필요한 버그
- 소셜 엔지니어링 공격
- 서드파티 의존성의 이슈 (업스트림에 보고해 주세요, 단 알려주시면 감사합니다)

## 보안 모범 사례

rdprrap은 다음 보안 관행을 따릅니다:

- 모든 `unsafe` 블록에 `// SAFETY:` 불변 조건 문서화
- 경계 확인 없는 원시 포인터 산술 금지
- DLL 로딩을 system32 경로로 제한 (`LOAD_LIBRARY_SEARCH_SYSTEM32`)
- 스레드 일시정지에 스냅샷 기반 열거 사용 (TOCTOU 경쟁 방지)
- 릴리스 전 `cargo audit` 실행

## 감사의 말

보안 취약점을 책임감 있게 공개해 주시는 보안 연구 커뮤니티의 노력에 감사드립니다. 유효한 보안 이슈를 제보한 기여자는 (허락을 받아) 릴리스 노트에서 감사의 말씀을 전합니다.
